## 六、进度更新规则

### 1. 状态定义

| 状态 | 含义 |
|---|---|
| `已规划` | 已进入版本规划或设计文档，但尚未交付 |
| `开发中` | 已开始实现或验证，但未达到交付标准 |
| `已完成` | 已进入当前发布能力，并有代码与验证支撑 |
| `阻塞中` | 有明确阻塞原因、影响范围与处理建议 |

### 2. 证据要求

| 状态 | 需要证据 |
|---|---|
| 🚧 开发中 | 代码文件路径 / 变更记录 / 验证记录 |
| ✅ 已完成 | 代码 + QA 结果 + 实机验证 |

### 3. 约束规则

- 讨论完成 ≠ 开发完成
- 文档完成 ≠ 代码完成
- 开发完成 ≠ 交付完成
- 每一项“已完成”都应能回答：代码在哪、怎么验证、边界是什么
