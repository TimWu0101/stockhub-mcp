# 借鉴与引用仓库清单

> 归档说明：本文为方案讨论阶段的历史参考记录，保留当时语境中的“后续/规划”表述，不代表当前项目实时状态；当前事实以 `docs/tracking/development-status.md` 与现行工具清单为准。

这份清单用于记录 `stockhub-mcp` 在方案讨论阶段明确借鉴、分析或参考过的 GitHub 仓库，方便后续继续查阅。

如只想先看 archive 的收敛结论，建议优先阅读 [`README.md`](./README.md) 与 [`skills-to-stockhub-mapping.md`](./skills-to-stockhub-mapping.md)，本文主要保留外部参考来源清单。

## 1. mcp-yfinance-server

- 仓库用途：基于 `yfinance` 提供 MCP 形式的美股/ETF/基础行情能力。
- 我们借鉴的点：
  - MCP 工具封装方式
  - `yfinance` 作为美股 / 中国香港股票默认数据源的可行性
  - 行情、历史数据、基础研究类接口的能力边界
- 适合承接：
  - 美股实时行情
  - 美股历史 K 线
  - ETF 基础数据
- 局限：
  - 对中国市场覆盖不足
  - 更偏单一数据源封装，不够支撑统一多市场架构

## 2. StockMCP

- 仓库用途：股票查询类 MCP 服务实现。
- 我们借鉴的点：
  - 面向 AI 的查询工具设计方式
  - 股票相关 MCP 接口组织形式
  - 输出结构如何更适合对话式调用
- 适合承接：
  - 工具命名风格
  - 查询参数组织
  - AI 调用体验参考
- 局限：
  - 覆盖范围相对有限
  - 不足以直接扩展为多市场统一金融数据后端

## 3. leek-fund

- 仓库用途：面向中国市场的基金 / 股票相关数据获取工具，内含多类公开接口调用思路。
- 我们借鉴的点：
  - 中国市场免费公开接口路线
  - A股 / 基金 / 板块 / 资金流相关接口来源
  - 通过公开网页接口获取数据的实现思路
- 适合承接：
  - A股实时行情与搜索
  - 基金与 ETF 数据
  - 板块、资金流等中国市场特有能力
- 备注：
  - 我们之前已明确参考过其中发现的免费接口思路，并据此设计中国市场主闭环。

## 4. yfinance-mcp

- 仓库用途：围绕 `yfinance` 的 MCP 封装实现。
- 我们借鉴的点：
  - `yfinance` 在 MCP 场景下的接口暴露方式
  - 美股 / 中国香港股票 / 基础基本面能力的组织方式
- 适合承接：
  - 美股与中国香港股票的默认基础能力
  - 历史价格、基础资料、部分基本面
- 局限：
  - 核心仍是单源模式
  - 中国市场深度能力不足

## 5. yahoo-finance-mcp

- 仓库用途：基于 Yahoo Finance 数据的 MCP 形式封装。
- 我们借鉴的点：
  - Yahoo Finance 数据在 AI 工具中的封装边界
  - 对 quote / history / company profile 这类通用能力的设计方式
- 适合承接：
  - 通用美股行情层设计
  - 返回 schema 参考
- 局限：
  - 无法单独满足 A股、基金、期货等中国市场需求

## 6. topnews-1.0.1

- 仓库用途：新闻聚合与分析型项目。
- 我们借鉴的点：
  - collectors 分层
  - 多源抓取
  - 并发采集
  - fallback 机制
  - 情感分析
  - 重要性评分
  - 板块 / 主题映射
  - 报告生成思路
- 适合承接：
  - 后续 `V1.1` 新闻与事件增强模块的架构参考
- 备注：
  - 更适合借鉴“新闻处理链路”，不是直接当作金融新闻主数据源。

## 7. tencent-news

- 仓库用途：新闻搜索 / 热点资讯类 MCP 能力封装。
- 我们借鉴的点：
  - 新闻查询入口设计
  - 热点 / 快讯查询形态
  - MCP / CLI 封装方式
  - 错误处理与输出规范
- 适合承接：
  - `get_market_news` 一类工具的交互形式参考
  - 通用新闻搜索层的能力设计
- 局限：
  - 更偏通用新闻，不是专门为金融市场设计
  - 缺少标的关联、事件结构化、市场分类等能力

## 8. AkShare（文档与生态）

- 参考地址：<https://github.com/akfamily/akshare>
- 我们借鉴的点：
  - 中国市场免费数据覆盖面
  - A股历史数据、基金 / ETF、期货研究数据、宏观数据承接能力
  - 免费优先前提下作为增强层和 fallback 的可行性
- 适合承接：
  - V0.2 中国市场增强版
  - V0.3 宏观与政策上下文版
- 备注：
  - 虽然我们重点查看过文档，但它本身也是后续实现中的关键参考仓库。

## 当前建议的使用定位

- **核心直接参考仓库**：`mcp-yfinance-server`、`StockMCP`、`leek-fund`、`yfinance-mcp`、`yahoo-finance-mcp`
- **新闻与资讯能力参考**：`tencent-news`、`topnews-1.0.1`
- **中国市场增强与兜底参考**：`AkShare`
